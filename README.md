# Multi-Agent Interview Coach

## Описание проекта

Данный проект представляет собой мультиагентную систему для проведения технического интервью. Система имитирует процесс реального интервью с кандидатом, используя несколько специализированных LLM-агентов (использовался mistral-large-latest), каждый из которых отвечает за свою роль (интервьюер, критик, менеджер и т.д.).

Ключевая цель проекта — продемонстрировать:

* агентную архитектур,
* адаптивное интервьюирование,
* контекстную память,
* скрытую рефлексию,
* устойчивость к нестандартному поведению кандидата.

Оркестрация агентов реализована с использованием LangGraph.

Примеры работы лежат в logs: есть тестовые логи, а также сгенерированные мной.

---

## Архитектура решения

### Основные агенты

* **InterviewerAgent** — задаёт вопросы кандидату, адаптируя их сложность и тематику.
* **CriticAgent** — оценивает ответы кандидата, выявляет пробелы и red flags.
* **HiringManagerAgent** — формирует фокус интервью и финальное решение.
* **IntentRouter** — классифицирует интенты кандидата (ответ, stop, встречный вопрос).
* **Role Reversal Responder** — обрабатывает встречные вопросы кандидата.

### Оркестрация с применением LangGraph

Интервью реализовано как граф состояний:

```
route_intent
   ├── stop → finalize
   ├── candidate_question → role_reversal → update_state
   └── normal_answer → critic → update_state → next_question → loop
```

Каждый шаг интервью — это один проход по графу.

## Ключевые свойства системы

### Скрытая рефлексия

После каждого ответа кандидата система выполняет внутреннюю рефлексию (оценка критика и менеджера), которая:

* логируется,
* не показывается пользователю напрямую,
* используется для выбора следующего шага интервью.

### Поддержка контекста

* система хранит историю вопросов и ответов;
* предотвращает повторение вопросов;
* учитывает предыдущие оценки при выборе следующего вопроса.

### Адаптивность

В зависимости от качества ответа:

* усложняет вопросы,
* упрощает или уточняет формулировки,
* может сменить тему.

### Робастность

Система устойчиво обрабатывает:

* ответы не по теме,
* встречные вопросы кандидата,
* досрочное завершение интервью,
* агрессивное или некорректное поведение.

## Формат логов

Каждый шаг интервью логируется и включает:

* вопрос интервьюера,
* ответ кандидата,
* `internal_thoughts` — скрытую рефлексию агентов,
* промежуточные решения,
* финальный отчёт.

---

## Итоговый отчёт

По завершении интервью система автоматически формирует отчёт, включающий:

* итоговую оценку кандидата (Hire / No Hire),
* анализ hard skills и soft skills,
* выявленные пробелы,
* рекомендации по развитию.
